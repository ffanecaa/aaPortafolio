WheGym

🡆

WheGym es una app que gestiona 2 tipos de perfil: El administrador de un gimnasio que pone a disposicion de la app su gymnasio y el cliente de un gymnasio que podra acceder a las instalaciones de los mismos y asistir a sus actividades por medio de una membresia. Existen diferentes membresias y estas generan un codigo QR personalizado con el que se podrá acceder a los mismo.

La implementación de esta aplicación se ha llevado a cabo a lo largo de cuatro sprints, con la finalización de los puntos de historia dentro del plazo establecido. Para el desarrollo del frontend, se ha empleado el framework Angular, utilizando el entorno de desarrollo Visual Studio Code. En cuanto al backend, hemos optado por el framework Sprint, utilizando la herramienta de desarrollo IntelliJ IDEA. Para la gestión de la base de datos optamos por postgreSql utilizado DBeaver

Retroshop

🡺

Esta aplicación web facilita la compra y venta de tecnología retro. Entre las funcionalidades implementadas se encuentran el registro de usuarios con verificación mediante correo electrónico, inicio de sesión, búsqueda por categoría y producto, gestión de perfiles con historial de transacciones, opción de marcado como favoritos y valoración de vendedores con el cálculo del promedio de estas evaluaciones.

Para el desarrollo del frontend, se ha utilizado la libreria React, mientras que para los estilos se ha optado por la herramienta Tailwind CSS. En cuanto al backend, se ha implementado con Express y se ha utilizado MySQL como sistema de gestión de bases de datos.

Galicia Monumental

🡺

Aplicaciones web cuyo objetivo principal es localizar más de 30,000 elementos patrimoniales de Galicia. Esto incluye todos los Bienes de Interés Cultural (BIC). La aplicación permite a los usuarios filtrar estos elementos por tipología, nombre y ubicación geográfica. Además, ofrece la funcionalidad de geolocalización, permitiendo al usuario ver los elementos visitables dentro de un radio de 30 km y calculando la ruta óptima para llegar a ellos. Aunque el proyecto actualmente cumple con los requisitos mínimos para ser funcional, es importante destacar que está inconclusa dado las dimensiones deeste proyecto.

Para el desarrollo del frontend, se ha utilizado el framework Svelte, conocido por su eficiencia y facilidad de uso, junto con Leaflet para la visualización de mapas interactivos. En cuanto al backend, se ha implementado con Express.js, un framework de Node.js ampliamente utilizado para la creación de APIs RESTful, y Sequelize se ha utilizado como ORM para la gestión de la base de datos, garantizando así una manipulación segura y eficiente de los datos.
